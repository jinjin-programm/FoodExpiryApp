Fix bitmap double-recycle at line 132 in DetectionPipeline.kt:

**Problem analysis:** At line 132, `detection.cropBitmap?.recycle()` recycles the bitmap. But `result` (created at lines 116-121 via `detection.copy(...)`) still holds a reference to the SAME bitmap object. `detection.copy()` is a shallow copy — it copies the reference, not the bitmap pixels. So recycling `detection.cropBitmap` also invalidates `result.cropBitmap`.

**Fix:** After creating the result (which has the updated status + foodIdentification), null out the cropBitmap on the result since it's no longer needed after LLM classification:

```kotlin
// Line 131-134 replacement:
// Recycle crop bitmap IMMEDIATELY per YOLO-08/PITFALL-6
detection.cropBitmap?.recycle()

// Per D-10: Null out cropBitmap on result to prevent stale reference
// The bitmap is recycled above; result should not hold a reference to it
results.add(result.copy(cropBitmap = null))
```

This ensures:
1. Bitmap IS recycled immediately (YOLO-08) ✓
2. Result doesn't hold stale reference to recycled bitmap (D-10) ✓
3. cropBitmap is already used by this point (LLM inference completed) ✓
