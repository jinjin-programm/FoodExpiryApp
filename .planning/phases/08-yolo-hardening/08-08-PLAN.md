**loadModel():**
1. Check `isLoaded`, return true if already loaded
2. Call `lifecycleManager.acquire(ModelType.YOLO)` — return false if can't acquire
3. Copy `yoloe-26n-seg-pf.onnx` from assets to a temp file (OrtSession needs file path for large models)
4. Create `OrtEnvironment.getEnv()` and `OrtSession.SessionOptions()`:
   - `opt.setOptimizationLevel(OrtSession.SessionOptions.OptLevel.ALL_OPT)`
   - `opt.addConfigEntry("session.intra_op.num_threads", "4")`
5. Create `OrtSession(environment, modelFilePath, options)`
6. Log input/output tensor shapes for debugging: `session.inputNames`, `session.outputNames`
7. Set `isLoaded = true`

**detect(bitmap: Bitmap): List<DetectionResult>:**
1. If not loaded, return emptyList()
2. Resize bitmap to 640×640 (simple scaling, no letterbox — YOLOE handles this)
3. Convert bitmap to NCHW float buffer: shape [1, 3, 640, 640]
   - Extract RGB pixels, normalize to 0.0-1.0
   - Arrange as [C, H, W] channels (R plane, G plane, B plane)
4. Create `OnnxTensor.createTensor(environment, floatBuffer, longArrayOf(1, 3, 640, 640))`
5. Run inference: `session.run(mapOf("images" to inputTensor))`
6. Extract output0 tensor: shape [1, 300, 38]
7. Per D-04/D-08, parse detections:
   - For each of 300 candidates:
     - Read first 4 values as bbox [x1, y1, x2, y2] in 640×640 pixel space
     - Read value at index 32 as confidence score
     - Read value at index 33 as class_id (integer)
     - If confidence < 0.5, skip (per D-08)
   - Scale bbox from 640×640 to original image dimensions
   - Take top-8 by confidence (per D-08, max 8 detections)
   - Apply NMS via `MnnYoloPostprocessor.applyNms()` with IoU threshold 0.45
   - Map label "class_{id}" using existing `mapLabelToCategory` pattern
8. Return `List<DetectionResult>` with normalized 0f-1f bounding box coordinates

**unloadModel():**
1. Close OrtSession and OrtEnvironment
2. Set isLoaded = false
3. Release via `lifecycleManager.release(ModelType.YOLO)`

**isModelLoaded():** Return isLoaded boolean.

Important implementation details:
- NCHW format is critical: ONNX models expect [batch, channels, height, width], NOT NHWC like TFLite
- The bitmap→NCHW conversion must be: for each channel (R, G, B), flatten all pixels in row-major order
- Normalize pixel values to [0.0, 1.0] by dividing by 255.0f
- Use `FloatBuffer.allocate(3 * 640 * 640)` for the input tensor
- Handle OrtException in detect() — catch and return emptyList() with Log.e

Per D-10: parseDetections needs adjustment for YOLOE output format.

The current `parseDetections()` assumes a flat row-based format `[x1, y1, x2, y2, confidence, class_id, ...]` with coordinates in 640×640 space. The YOLOE model output is `(1, 300, 38)` where each row has:
- Indices 0-3: bounding box [x1, y1, x2, y2] in 640×640 pixel space
- Indices 4-35: mask coefficients (32 values, NOT USED)
- Index 36: confidence score
- Index 37: class_id

Update `parseDetections()`:
1. Change the confidence index from `base + 4` to `base + 32` (per YOLOE format — verified: index 32 is the score in the 38-value row, but the exact position may vary. Use `base + numValues - 2` for score and `base + numValues - 1` for class_id as a fallback)
2. Actually, let's be safe: add a new overloaded function `parseYoloEDetections()` that specifically handles the (300, 38) format:
   - Confidence at index 36 (second to last)
   - Class ID at index 37 (last)
   - BBox at indices 0-3
3. Keep the original `parseDetections()` unchanged for backward compatibility (MnnYoloEngine may still reference it)
4. `applyNms()` and `cropDetection()` need NO changes — they work with normalized DetectionResult coordinates regardless of model format
5. `mapLabelToCategory()` works with "class_{id}" labels — no change needed

Add the new function:
```kotlin
fun parseYoloEDetections(
    outputArray: FloatArray,  // Flattened (300, 38) tensor
    numDetections: Int,        // 300
    numValues: Int,             // 38
    originalWidth: Int,
    originalHeight: Int,
    confidenceThreshold: Float = 0.5f,
    maxDetections: Int = 8
): List<DetectionResult>
```

This function:
1. Iterates 0..numDetections (300 candidates)
2. For each: base = i * numValues
3. Extracts bbox [x1,y1,x2,y2] from indices 0-3
4. Extracts score from index numValues-2 (36)
5. Extracts class_id from index numValues-1 (37)
6. Filters by confidenceThreshold
7. Sorts by confidence descending, takes maxDetections
8. Normalizes bbox from 640×640 to 0f-1f (same logic as existing parseDetections)
9. Returns List<DetectionResult>
