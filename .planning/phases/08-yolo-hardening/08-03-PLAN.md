**Part A: Add detections StateFlow to YoloScanViewModel**

Add to YoloScanViewModel:
```kotlin
private val _detections = MutableStateFlow<List<DetectionResult>>(emptyList())
val detections: StateFlow<List<DetectionResult>> = _detections.asStateFlow()
```

In `startDetection()` collect lambda, update detections when PipelineState changes:
```kotlin
is PipelineState.Classifying -> {
    // First Classifying emission means YOLO completed — save detections for overlay
    if (_detections.value.isEmpty()) {
        // Detections are available but we need access to raw YOLO results
        // The pipeline doesn't emit raw detections, only the Classifying count
        // Solution: We need to modify the pipeline to emit raw detections
        // OR: Track detections separately in the ViewModel via a repository query
    }
    YoloScanUiState.Classifying(state.current, state.total)
}
```

**Better approach:** Modify DetectionPipeline to also emit raw YOLO detections as part of Classifying state, OR add a new state `PipelineState.YoloComplete(val detections: List<DetectionResult>)` between Detecting and Classifying.

Simplest approach that doesn't require pipeline changes: Store the bitmap in ViewModel, and when `Detecting` transitions to `Classifying`, the YOLO results are already available as part of the pipeline's internal state. But since the pipeline runs in a channelFlow, we can't easily access intermediate state.

**Chosen approach:** Add a new PipelineState subclass:
```kotlin
data class YoloComplete(val detections: List<DetectionResult>) : PipelineState()
```
Emit this in DetectionPipeline after YOLO detection and before LLM classification starts. This gives the fragment immediate access to raw detections for overlay rendering.

In YoloScanViewModel:
```kotlin
is PipelineState.YoloComplete -> {
    _detections.value = state.detections
    YoloScanUiState.Detecting // Still show "Detecting" until Classifying starts
}
```
