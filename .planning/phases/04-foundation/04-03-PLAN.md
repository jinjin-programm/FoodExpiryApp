    MNN AAR and native libraries need to be obtained. The agent cannot build MNN from source or download from external sources.
  </what_built>
  <how_to_verify>
    **Option A: Download Prebuilt AAR (Recommended)**
    
    1. Check MNN GitHub releases for prebuilt Android AAR:
       https://github.com/alibaba/MNN/releases/tag/3.5.0
    
    2. If available, download the Android AAR file.
    
    **Option B: Build from Source**
    
    1. Clone MNN repository:
       ```
       git clone https://github.com/alibaba/MNN.git
       cd MNN
       git checkout 3.5.0
       ```
    
    2. Build for Android (requires Android NDK):
       ```
       cd project/android
       mkdir build_64
       ../build_64.sh -DMNN_BUILD_LLM=true -DMNN_OPENCL=true -DMNN_USE_LOGCAT=true
       ```
    
    3. The output AAR will be in: `build_64/output/MNN-release.aar`
    
    **After obtaining the AAR:**
    
    4. Create the libs directory:
       ```
       mkdir -p app/libs
       ```
    
    5. Copy the AAR:
       ```
       cp /path/to/MNN-release.aar app/libs/
       ```
    
    6. Extract native libraries from the AAR (it's a ZIP file):
       - Open the AAR as a ZIP
       - Navigate to `jni/arm64-v8a/`
       - Copy `libMNN.so` and `libllm.so` to `app/src/main/jniLibs/arm64-v8a/`
    
    7. Verify files exist:
       ```
       ls -la app/libs/MNN-release.aar
       ls -la app/src/main/jniLibs/arm64-v8a/libMNN.so
       ls -la app/src/main/jniLibs/arm64-v8a/libllm.so
       ```
  </how_to_verify>
  <resume_signal>Type "done" when MNN AAR and native libraries are in place</resume-signal>

  </read_first>
  <action>
    **Step A: Add MNN AAR dependency to build.gradle.kts**

    In `app/build.gradle.kts`, in the `dependencies` block, ADD this line (after the existing dependencies):

    ```kotlin
    // MNN Inference Engine (v3.5.0)
    implementation(files("libs/MNN-release.aar"))
    ```

    **Step B: Add ProGuard rules for MNN**

    In `app/proguard-rules.pro`, ADD the following rules at the end:

    ```proguard
    # MNN Inference Engine
    -keep class com.alibaba.mnn.** { *; }
    -keepclassmembers class com.alibaba.mnn.** { *; }
    
    # MNN native methods
    -keepclasseswithmembernames class * {
        native <methods>;
    }
    ```

    **Step C: Verify jniLibs directory structure**

    Ensure the native libraries are in the correct location:
    ```
    ls -la app/src/main/jniLibs/arm64-v8a/
    ```
    Should show: `libMNN.so`, `libllm.so` (and possibly other MNN libs like `libMNN_CL.so` for OpenCL).

    **Step D: Verify NO cmake configuration remains**

    The `externalNativeBuild` block should have been removed in Plan 02. Verify:
    ```
    grep -n "externalNativeBuild\|cmake" app/build.gradle.kts
    ```
    Should return no matches (or only comments).

    **Step E: Clean and rebuild**

    ```
    ./gradlew clean
    ./gradlew assembleDebug
    ```

    The build should succeed. If there are errors about missing MNN classes, the AAR may not be correctly referenced. Check that:
    - `app/libs/MNN-release.aar` exists
    - The path in `implementation(files(...))` matches the actual file location
  </action>

  </verify>

    - ProGuard rules for MNN added

  </done>
