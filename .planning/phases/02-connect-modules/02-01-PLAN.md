    ```
    sortedByDescending { it.matchCount * 10 + it.urgencyLevel * 5 + it.wasteRescuePercent }
    ```
    
    Change the scoring to give significant weight to items expiring within 7 days:
    - Items already expired: weight 10
    - Items expiring in 1 day: weight 8
    - Items expiring in 2-3 days: weight 5
    - Items expiring in 4-7 days: weight 3
    - Items expiring in 8+ days: weight 1
    
    In `ScoreRecipesForInventoryUseCase.invoke()`, calculate an `expiryUrgencyBonus` per matched inventory item based on `daysUntilExpiry`, then add it to the sort score. This ensures recipes that rescue soon-to-expire items appear at the top.
    
    In `RecipesViewModel`, the `applyFilter` for `RecipeFilter.ALL` currently just returns all matches. Add a "promoted" section concept: recipes with any matched items expiring within 7 days should be sorted to the top of the default view. The `expiringItems` field already filters `daysUntilExpiry <= 3` — extend this to `<= 7` to feed the scoring algorithm.
    
    Do NOT change the data model. Do NOT add new UseCase classes. Only modify the scoring logic and ViewModel filtering.
